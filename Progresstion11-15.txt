Progression13: Handling + DOM Manipulation + CRUD (Create/Delete)
คุณได้ออกแบบระบบ Quote Management โดยแยกโค้ดออกเป็น 3 Layer อย่างเป็นระเบียบ:

Fetch Utilities

UI Logic & DOM
จัดการ event, สร้าง DOM element ของ quotes, update UI แบบ real-time เมื่อ delete หรือ add สำเร็จ

Progression15: สร้างระบบให้สามารถ กดปุ่ม “Change” เพื่อประกาศหรือแก้ไข Study Plan โดยใช้ async function แยกตาม HTTP method (POST/PUT)
สร้าง postDeclaredPlan และ putDeclaredPlan สำหรับสร้างและแก้ไข plan
สร้าง declarePlan(studyPlans, isUpdate) เพื่อจัดการ logic ทั้งหมด:
เลือก function request ตาม isUpdate (POST หรือ PUT)
ตรวจสอบ plan ที่เลือกจาก dropdown
ส่ง request ไป API และจัดการ response ด้วย try/catch
การจัดการ response:
200 OK (PUT) → merge ข้อมูล, แปลงเวลาเป็น local, update UI, แสดง modal success
404 Not Found → แสดง modal ว่าไม่พบ declared plan, รีโหลด UI
201 Created (POST) → รีโหลด UI หลังสร้าง plan ใหม่
409 Conflict → แสดง modal ว่าเคยdeclaredแล้ว, รีโหลด UI
500/502/503 → แสดง modal แจ้งบอก server error
กรณีerrorอื่น ๆ → alert error
